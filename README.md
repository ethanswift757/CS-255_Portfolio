# CS 255 Portfolio

## DriverPass Project Summary
The DriverPass project was based on a client who wanted an online system to help students prepare for their driver's license exam. The system allows customers to create accounts, purchase training packages, schedule driving lessons, take practice tests, and view test results. Administrators and trainers also have features to manage customers, schedules, training packages, and lesson status.

## What I Did Well
I think I did a good job identifying the customer's requirements and turning them into UML diagrams that clearly showed how the system should work. Creating the use case and activity diagrams helped me organize the different parts of the system and understand how each user interacts with DriverPass.

## What I Would Improve
If I could revise one part of the project, I would improve some of my UML diagrams by making them more detailed and organized. After completing the project, I have a better understanding of how to represent different system processes and relationships.

## Understanding User Needs
I focused on the client's request for a system that is simple to use while allowing customers, trainers, and administrators to perform their tasks efficiently. Considering the user's needs is important because a system can have many features, but if it is difficult to use or does not solve the customer's problem, it will not be successful.

## My Software Design Approach
When designing software, I like to begin by understanding the client's requirements before creating diagrams and planning the system. Breaking the project into smaller parts with UML diagrams made the design easier to understand. In future projects, I will continue using use case, activity, and sequence diagrams to organize requirements before moving into development.
